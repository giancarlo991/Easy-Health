🗣️ Roteiro Pitch de Apresentação (Máximo 5 Minutos)

Easy Health — Integração e Entrega Contínua (Prof. Silvio Barbieri)

⚡ Estratégia do Pitch de 5 Minutos:

Apresentação extremamente ágil. Fale com ritmo firme, não se prenda a detalhes de telas, mostre os arquivos de teste e segurança do código apenas por cima e ressalte que o fluxo é 100% automatizado no GitHub Actions.

📌 Bloco 1: Introdução e Proposta (Slides 0 e 1) ⏱️ 45 segundos

🎙️ André: "Boa noite. O Easy Health é uma plataforma voltada para Personal Trainers e Nutricionistas gerenciarem seus alunos e treinos de forma integrada. Nosso objetivo em Engenharia de Software foi construir uma aplicação dividida em micro-serviços (Front e Back) com qualidade de código assegurada por uma esteira automática de CI/CD em GitHub Workflows, cobrindo compilação, testes e monitoramento."

📌 Bloco 2: Stack e Infraestrutura (Slides 2 e 3) ⏱️ 1 minuto

🎙️ Daniel: "Nossa stack conta com Frontend em React via Vite e Backend em Express com banco documental MongoDB via Mongoose, garantindo flexibilidade para tabelas complexas de treinos. No quesito infraestrutura, criamos arquivos Docker Compose para orquestrar o banco localmente. Nossa esteira do GitHub Actions valida a compilação do código e o padrão estático com ESLint a cada push, disparando deploys automáticos diretamente para a Vercel."

📌 Bloco 3: Integração e Comunicação (Slide 4) ⏱️ 1 minuto

🎙️ Gustavo: "A comunicação entre as partes é feita por chamadas de API centralizadas via Axios. Criamos um interceptador no frontend que insere automaticamente o Token JWT nas requisições seguras. Caso a sessão expire (retornando status HTTP 401), o próprio interceptador limpa as credenciais locais e redireciona o usuário para a tela de login, garantindo robustez de navegação."
Destaque no código: Diga que o arquivo principal de integração é o api.js do React.

📌 Bloco 4: Testes, Segurança e Logs (Slides 5 e 6) ⏱️ 1.5 minutos

🎙️ Gustavo: "Para blindar a segurança e qualidade, implementamos testes em duas camadas no pipeline de CI/CD: testes unitários com Jest mockando o banco no AuthService (registro e login) e testes de integração com Newman rodando endpoints HTTP contra um MongoDB em container Docker no GitHub Actions. Em segurança, as senhas usam criptografia irreversível com Bcrypt de 10 rounds de salt e rotas privadas protegidas por JWT. Por fim, criamos monitoramento de visualização de perfis semanais persistido no banco."
Foco em Testes: Reforce que Jest roda em milissegundos e Newman garante o fluxo fim a fim no Actions.

📌 Bloco 4.5: Evolução do Pipeline (Bugs no Actions) ⏱️ 45 segundos

🎙️ Giancarlo: "Durante o setup do CI, tivemos 5 execuções de testes. Corrigimos um problema clássico de incompatibilidade de ambientes: imports que funcionavam no Windows local quebravam no runner Linux do GitHub Actions devido à sensibilidade de maiúsculas/minúsculas (Case-Sensitivity) nos nomes de arquivos. Ajustamos os caminhos dos models nos commits #2, #3 e #4, e no commit final dividimos os testes em jobs independentes com timeout de inicialização da API corrigido com wait-on, garantindo sucesso total."

📌 Bloco 5: Resultados e Conclusão (Slides 8 e 9) ⏱️ 45 segundos

🎙️ Giancarlo: "Concluímos preenchendo todos os requisitos da planilha de P2. Nossa esteira assegura build, testes unitários, testes de integração e deploy automático. O projeto está implantado em produção e funcional, seguindo boas práticas de DevOps e Segurança de dados. Agradecemos aos professores e abrimos espaço para perguntas."
